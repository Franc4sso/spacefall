# La Fame (Hunger) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Trasformare il cibo in una pressione di sopravvivenza quotidiana — fame per personaggio (visibile), carte a gradini (opportunità → razionamento/triage → sacrificio), fonti di cibo guidate dagli oggetti, il tutto bilanciato col simulatore.

**Architecture:** Backend-first. Aggiungere un campo `hunger` per personaggio, estendere l'effetto `character` e i selettori, una condizione `crew_hunger`, e una tappa "fame" nella pipeline giornaliera (`DayProcessor`). Poi contenuto narrativo (le famiglie di carte) gated su soglie di `food`/`hunger` con `weight_modifiers` per la cadenza. Infine UI (fame sugli avatar) e tuning col simulatore. Riusa il sistema "equipaggio vivo" (standing/reazioni/flag-testimone/crew_trust) per le conseguenze.

**Tech Stack:** PHP 8.3 / Laravel 12 / SQLite · Pest (TDD backend) · React 19 / TypeScript / Vite (verifica `npx tsc --noEmit`).

**Convenzioni chiave:**
- `hunger` vive dentro ogni elemento dell'array `characters` (come `stress`), intero 0–100, default 0.
- I numeri di tuning vivono in `config/game.php` sezione `hunger` e nel `daily` di `food`; i valori iniziali qui sono punti di partenza, **tarati nel Task 9** col simulatore.
- Morte per inedia = stato accumulato visibile nel `DayProcessor` (NON una scelta-trappola), così `BalanceTest` (morti-da-scelta evitabili) resta verde.

- [ ] **Step 5: Implementa `applyHunger`**

In `backend/app/Game/DayProcessor.php`, aggiungi il metodo privato (vicino a `applyHardship`):

```php
    /**
     * Daily hunger: every living survivor gets hungrier; above the configured
     * bands hunger inflicts stress; at starve_at the survivor dies. Returns the
     * updated roster and whether anyone starved (for the witness flag).
     *
     * @param  list<array<string,mixed>>  $characters
     * @return array{0: list<array<string,mixed>>, 1: bool}
     */
    private function applyHunger(array $characters): array
    {
        $cfg = config('game.hunger');
        $rise = (int) ($cfg['daily_rise'] ?? 0);
        $starveAt = (int) ($cfg['starve_at'] ?? 100);
        $bands = $cfg['stress_bands'] ?? [];
        $someoneStarved = false;

        foreach ($characters as $i => $c) {
            if (! ($c['alive'] ?? true)) {
                continue;
            }
            $hunger = $this->clampResource((int) ($c['hunger'] ?? 0) + $rise, 100);
            $characters[$i]['hunger'] = $hunger;

            if ($hunger >= $starveAt) {
                $characters[$i]['alive'] = false;
                $someoneStarved = true;
                continue;
            }

            // Highest matching stress band applies.
            $add = 0;
            foreach ($bands as $band) {
                if ($hunger >= ($band['at_or_above'] ?? 0)) {
                    $add = max($add, (int) ($band['stress'] ?? 0));
                }
            }
            if ($add > 0) {
                $characters[$i]['stress'] = $this->clampResource((int) ($c['stress'] ?? 0) + $add, 100);
            }
        }

        return [$characters, $someoneStarved];
    }
```

```php
<?php

use App\Game\Sim\GreedySurvivalPolicy;
use App\Game\Sim\Simulator;
use Database\Seeders\ContentEventSeeder;
use Database\Seeders\EventSeeder;

beforeEach(function () {
    $this->seed(EventSeeder::class);
    $this->seed(ContentEventSeeder::class);
});

function medianDay(Simulator $sim, $policy, array $items, int $count): int
{
    $days = [];
    for ($i = 0; $i < $count; $i++) {
        $days[] = $sim->play($i, $policy, $items)->day;
    }
    sort($days);
    return $days[(int) floor(count($days) / 2)];
}

it('a food-poor crew is pressured but not instantly doomed', function () {
    $sim = app(Simulator::class);
    // No food items in the kit: hunger must bite, but a careful player should
    // not collapse in the first handful of days.
    $median = medianDay($sim, new GreedySurvivalPolicy(), ['welder', 'scanner', 'comms', 'medkit', 'manual'], 60);
    expect($median)->toBeGreaterThanOrEqual(10);
});

it('a food engine lets a skilled crew sustain itself longer', function () {
    $sim = app(Simulator::class);
    $poor = medianDay($sim, new GreedySurvivalPolicy(), ['welder', 'scanner', 'comms', 'medkit', 'manual'], 60);
    $fed = medianDay($sim, new GreedySurvivalPolicy(), ['seedbank', 'rations', 'drone', 'rifle', 'medkit'], 60);
    // Investing the loadout in food must measurably extend survival.
    expect($fed)->toBeGreaterThan($poor);
});
```

## Note per l'esecutore

- I metodi-fame (`hungerOpportunityEvents`/`hungerRationEvents`/`hungerCrisisEvents`) vanno aggiunti dentro `ContentEventSeeder` e inclusi nell'`array_merge` di `events()`.
- `hunger` vive SEMPRE dentro l'array `characters` (come `stress`); l'effetto `character` con chiave `hunger` lo muove; la condizione `crew_hunger` lo legge.
- La morte per inedia avviene nel `DayProcessor` (end-of-day), quindi conta come morte NON-da-scelta per `FairnessProbe` — coerente con `death_starvation`. L'unica morte da-scelta nuova è il sacrificio, che ha sempre un'alternativa sopravvivibile.
- Il tuning (Task 9) è parte del lavoro, non un optional: i numeri iniziali sono punti di partenza, gli obiettivi misurabili sono nel test.
- `EventSchema` non valida le chiavi degli effetti `character` (stress/hunger sono campi liberi dell'effetto), quindi `hunger` negli outcome non richiede modifiche allo schema oltre alla condizione `crew_hunger`.
